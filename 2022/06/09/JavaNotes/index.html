参考学习

基本概念

  1. JDK(Java Development Kit ,java开发工具包）
    开发用；

  2. JRE (Java Runtime Environment ,Java运行时环境)
    运行用；

  3. JVM(Java Virtual Machine,Java虚拟机)
    核心；

  4. bin 中放的是.class 文件

  5. src 中放的是 源代码

  6. 包(完全小写)cn.itcast.day.demo —–> cn\itcast\day\demo

  7. 所有的类都需要导包才能使用除了两种情况：
    1.要使用的类，与当前类位于同一个包下；
    2.要使用的目标类，直接位于java.lang包下，但是不包含子包

eclipse 常用快捷键：

格式化：ctrl+shift+f ;
智能提示：alt+/ ;
添加单行注释：ctrl+/ ; 取消：ctrl+shift+/ ；
添加多行注释：全选后 ctrl+shift+/ ；取消：ctrl+shift+\ ；
向下复制一行：ctrl+alt+向下箭头 ；
向上复制一行：ctrl+alt+向上箭头 ；
向上/下移动一行代码：alt+向上箭头/向下箭头 ；
删除当前行：ctrl+d ;
统一重命名：鼠标选中+alt+shift+r;

配置环境变量：
  1. JAVA_HOME : java安装路径；
  2. path:%JAVA_HOME%\bin;(最好放在第一个)

%JAVA_HOME%：引用JAVA_HOME路径

注释：
  1. 单行注释：//
  2. 多行注释：/* */
  3. 文档注释：/** javadoc */
cmd命令提示符：
  1. 进入磁盘 : name：
  2. 进入文件夹: cd name
    (tab 可找文件名)
  3. 回退至上一个文件夹: cd ..
  4. 回至根目录: cd \
  5. 查看当前文件内容: dir
  6. 清理cmd内容: cls
  7. 退出cmd: exit
变量命名规则：
  1. 类：每个单词首字母大写
  2. 变量/方法：第一个单词完全小写，后续单词首字母大写
  3. 包：全部小写，多级包用点隔开
数据类型：

  1. 基本类型：
    整数类型：byte short int long 字节数：1 2 4 8
    浮点类型：float double 字节数：4 8
    字符类型：char 字节数：2
    布尔类型：boolean 字节数：1 (只能是true/false,不支持1/0）
    定义一个long类型后面加L
    定义一个float类型后面加F
    不加L/F则为int/double类型

    tips:
    常量：public static final 数据类型 NAME=x;
    字符常量 char :有且仅有一个字符（不能为空串‘’错误写法）
    空常量null不能直接打印 指示没有对象 使用==判断
    比较浮点数，不能判断相等，检查接近程度
    强制类型转换：
    double balance
    int dollars
    int dollars = (int)balance

  2. 引用类型：
    1.字符串: String ；
    2.数组：数据类型[] 数组名称 ；
    数组要想使用，要先初始化：

    1. 动态初始化，指定数组的长度：
      数据类型[] 数组名称 = new 数据类型[数组长度];
      new 代表创建动作，内存中开辟空间，创建数组；
    2. 静态初始化，指定数组的内容；(自动推算数组长度)

  3. 标准：数据类型[] 数组名称 = new 数据类型[] {元素1,元素2，元素3,….};

  4. 简便：数据类型[] 数组名称 = {元素1,元素2，元素3,….};
    获取数组长度：arryName.length;
    数组索引从0开始没有负值；

构建实例变量时：
String 或其他引用类型 默认赋值null;
整数 默认赋值0；
浮点数 默认赋值0.0；
char 默认赋值‘\u000’;
Boolean 默认赋值 false;

==比较：
对于基本类型是进行数据内容的比较；
对于引用类型是进行地址的比较；

使用变量的注意事项：

1.创建的多个变量不能重名；
2.变量如果没有赋值，不能直接使用；
3.变量的作用域问题；
作用域：变量定义在哪个大括号中就只能在哪个大括号内使用
4.支持int a,b,c / int a = 10,b = 20的定义方法

计算：
  1. 有小数参与计算 结果一定为小数；x/10.0
  2. 只能对整数进行取模(%);
  3. 任何数据类型和字符串进行连接操作，结果都是字符串;
  4. 字符串连接时：
    注意先后顺序，优先级问题
    “hello”+20+10 结果为“hello2010”；
    “hello”+(10+20) 结果为“hello30”；
    “hello”+10*20 结果为 “hello200”；
  5. 自增运算符：
    混合使用时：
    前++ 先加后用 ；
    后++ 先用后加；
    如 ：num = 1
    System.out.println(++num) —-> 2;
逻辑运算符：

与 &
或 |
异或 ^ （相同为 false 不同为 true）
非 ！

短路使用：
短路与 &&；
短路或 ||；
如果表达式左边已经可以确定最终结果，那么右边的代码不再执行
如 10<2 && 10>= 10 —>false;
不执行判断10 >=10
凡是用到了与、或这两个逻辑运算推荐短路使用

Java 内存分配：

  1. 栈(Stack):主要用来存放局部变量；

  2. 堆(Heap):存放对象（new）；
    堆中的数据有默认值规则：
    1).整数 默认赋值0；
    2).浮点数 默认赋值0.0；
    3).char 默认赋值‘\u000’;
    4).Boolean 默认赋值 false;
    5).String 或其他引用类型 默认赋值null(空常量);

  3. 方法区(Method Area)：存放与.class（字节码文件）相关的信息；

  4. 本地方法区(Native Method Area):与操作系统相关；

  5. 寄存器(pc Register):与CPU相关；

方法(函数)：

  1. 定义方法：
    public static 返回值类型 方法名称(参数类型 参数名称,参数类型 参数名称,….)
    {
    方法体
    return 返回值;
    }

    Tips:
    修饰符： public static
    不能定义在main中；
    1).多个方法之间定义的前后顺序无所谓；
    2).不能在一个方法的内部嵌套定义另一个方法；
    3).变量和方法可以重名；
    4).返回值只能有一个/没有；

  2. 方法重载(Overload)：方法名称相同，参数列表不同
    对于类似功能的多个方法，只要记住一个相同的方法名，可以根据参数列表的不同自动适配
    参数列表不同：
    1).参数个数不同；
    2).参数类型不同；
    3).参数的多类型顺序不同；

    与参数名称无关；
    与返回值类型无关；

  3. 参数传递：
    1.对于基本数据类型以及String来说，形式参数的操作不会影响实际参数；
    2.对于引用数据类型除了String来说，形式参数的操作会影响实际参数；
    (相当于c++中的地址传递

    1.所有成员变量都使用private私有化；
    2.为每个成员变量编写 一对Setter Getter方法；
    3.编写一个无参数的构造方法；
    4.编写一个全参数的构造方法；

构造方法：

专门用来创建对象的方法，当new关键字创建对象时就调用构造方法

  1. 不能写返回值类型/void；
  2. 构造方法的名称必须和所在类的名称完全相同；
    public name(参数类型 参数名称,….){
    方法体：
    //return ;// 通常return 省略不写
    }
构造方法重载:
  1. 定义一个无参数的构造方法，可以直接new对象；
  2. 定义一个重载的全参数的构造方法，可以在new对象的同时方便对成员变量赋值

    ps:
    1.构造方法如果没有自定义，则编译器默认自动创建一个无参数构造方法，
    若自定义了一个构造方法，编译器则回收自动创建的构造方法，
    要重写无参数的构造方法;
    2.调用方法时，若对象名当中存储的是null值，那么会发生“空指针异常”

局部变量与成员变量区别：
1.定义位置不一样：
局部变量：定义在方法内部 （方法的参数也属于局部变量
成员变量：直接定义在类当中，在方法的外面；
2.在内存中的位置不一样：
局部变量：在栈内存当中；
成员变量：在堆内存当中；
3.生命周期不同：
局部变量：随着方法的进栈而出现，随着方法的出栈而消失；
成员变量：随着对象被创建而出现，随着对象被JVM回收而消失；
4.默认值不同：
局部变量：无默认值，必须赋值后使用；
成员变量：如果未赋值，则有默认值；

字符串：一个对象

字符串常量也是一个对象
字符串对象一旦被创建，内容就不能被改变

  1. 创建方法：
    1.String str1 = new String(char[] array);
    2.String str2 = new String(char[] array,offset,count);
    3.String str3 = “hello world”;

  2. 字符串池：(节省内存，重复利用字符串)
    1.位于堆内存中保存若干个字符串的地址值；
    2.不会出现重复的字符串对应的地址；
    3.凡是直接双引号赋值的字符串默认都在池中；而new出来的字符串默认不在池中；

  3. 字符串比较：
    1.==运算符：地址值的比较；
    2.字符串内容比较：

    比较原则：
    所有大写字母在小写字母之前；
    空格在所有可打印字符之前；
    数字在字母之前；
    public boolean equals(Object obj) 严格区分大小写
    String str1 = “abc”;
    String str2 = “bcd”;
    String str3 = “Abc”;
    boolean same = str1.equals(str2);
    str1.equalsIgnoreCase(str1); // true 忽略大小写判断

tips:
如果有一个字符串是常量则将常量写在前面；
“abc”.equals(str1);

字符串常用方法

  1. 字符串替换方法：
    public String replace(CharSequence oldStr,CharSequence newStr)

  2. 字符串切割方法：
    public String[] split(String regex)
    ps: 不要使用英文.作为分隔符，会与正则表达式冲突
    使用 \. 转义为 .

  3. public boolean string1.compareTo(string2) 比较字符串间顺序

  4. public char charAt(idx) 从字符串中返回一个char值

  5. public String[] substring(start,end) 从字符串中返回一个子串
